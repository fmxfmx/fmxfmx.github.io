Taptic-Engine 震动反馈

发表于 2019-05-23 | 分类于 iOS开发

Taptic-Engine 是苹果在2015年发布 iPhone6S 时的一个黑科技，现在的 iPhone 设备上的震动都是由这个东西做的。

并且在 iOS10 的时候推出这么一个类 UIFeedbackGenerator 我们可以使用它来做一些震动反馈，从而在一些特定的场景下提升用户的体验。下面我们就来看看这个类的使用

UIFeedbackGenerator

官方文档给出的解释是

The abstract superclass for all feedback generators.
所有 feedback generators 的父类

系统为我们提供了以下几个子类：

  • UIImpactFeedbackGenerator

作为某种反馈来提示用户，例如当交互界面中某个控件达到临界条件的时候，需要这么一个反馈来提示用户

  • UISelectionFeedbackGenerator

作为一个选择反馈来提示用户，例如当某个开关开启或者关闭的时候，可以使用这种子类去反馈用户

  • UINotificationFeedbackGenerator

做为一个通知的反馈来提示用户，并且该类包含了 successes failures warninigs

官方文档也为我们提供了一个例子

这里需要讨论下一个点就是 UIFeedbackGenerator 为我们提供了 -(void)prepare 方法，这个比较有意思，在这里说下

当我们调用这个方法时，generator 将会在很短的时间内进入一个准备状态，当 generator 准备完毕时，你可以很快的获得一个震动反馈，这个时间可以忽略不计。
我们会在需要反馈前去调用 prepare 方法，因为这样可以使我们在调用反馈时能够很快的起到作用，但是文档中提到如果我们调用完 prepare 时立即调用反馈，这样并不会改善反馈延迟。当我们不再需要到用到反馈时，应该将其移除引用，让系统 deallocate 掉它，这样就会使 Taptic Engine 回到一个空闲状态。

由于 UIFeedbackGenerator 适用于 iOS10 以上的设备 那么 iOS10 以下的设备可以使用这种方式来进行震动反馈

AudioServicesPlaySystemSound(SystemSoundID inSystemSoundID)

传入的 SoundID 有

经过测试，我们一般的震动来自于 kSystemSoundID_Vibrate 这个 SoundID ，但是我们可以通过这几个值来获取到不同的效果

// 普通短震，3D Touch 中 Peek 震动反馈
AudioServicesPlaySystemSound(1519);

// 普通短震，3D Touch 中 Pop 震动反馈
AudioServicesPlaySystemSound(1520);

// 连续三次短震
AudioServicesPlaySystemSound(1521);

但是这里明显是调用了私有的 API 官方不建议我们在发布的应用中这么去使用

除此之外，还有一种骚操作可以获取到反馈，这里也不再做再多的介绍了，因为调用的还是私有的 API
