如何搭建个人博客

发表于 2018-06-24 | 分类于 Life

本文就自己如何搭建博客经历做一个记录，现在搭建博客主要是有三种方式:

  • Wordpress
  • GitHub + Jekell
  • GitHub + Hexo

第一种方式可以自己购买独立的域名进行搭建，第二种方式比较麻烦，我最开始的时候就是使用的，为了搭建本地环境还是花了不少时间的，最后一种方式应该是最简单的了，所以我现在换成了最后一种搭建方式。

Hexo

这是一个基于 Nodejs的静态博客网站生成器，搭建本地环境也非常的简单，接下来就简单介绍下如何在Mac中基于 Hexo 搭建个人博客。

  1. 创建 GitHub 源
    这一步中所创建的源名字一定要是 用户名.github.io

  2. 下载 Nodejs
    下载好后双击安装，等待安装完成。

  3. 前往 Hexo 主页

在终端中输入主页中的命令即可安装(这一步需要cd到你想装载的目录下执行)

  1. 安装完成后，会自带一个默认的主题，在终端中键入 hexo s 根据提示就可以打开带有默认主题的博客了，里面也有一篇默认的 Hello world 的文章。

执行完这四步以后，整个环境就已经搭建完毕了，接下来介绍一些配置相关的内容

整个文件的目录如下:

  • source 文件夹中存放着一些资源文件，博客中的文章，关于页面，分类页面的资源都存放在这个文件中，后面会有介绍。
  • themes 下载的主题文件都会存在在这个文件夹中。
  • _config.xml 博客的整体配置文件，接下来会介绍这个文件。(一共会有两个_config.xml文件，在themes 文件夹中的单个主题文件夹还存在一份主题的_config.xml 配置文件)
  1. 如果需要自定义主题的，可以去 Hexo 官方文档中找到你喜欢的主题，在终端中执行 git clone 对应主题地址 你就会在 themes 文件夹中找到对应的主题了

整体 _config.xml

这里主要说明的是整个博客的配置文件，我只列出重要的部分

除了这几个需要我们配置以外，其余的配置就保持不变好了，官方文档中也有对用的解释，可以去了解下。

主题 _config.xml

这个配置文件主要是针对博客所选用的主题进行一个配置，内容不一定都相同，同样也只是列出重要的部分，我使用的是入门级的 next 主题，这款主题也是相当火的。

别的东西都是博客的属性了，根据每个人的喜好做一些不同的改变。这里主要说一下这几个菜单，因为之前有说到 sources 文件夹中用来存放一些关于页面，分类页面等内容，在刚刚创建出博客的时候，sources 文件夹中是没有这几个文件夹的，需要我们手动的去创建。

在终端中输入 hexo new page 页面名字 ,命令执行完后会创建出对应的文件夹，文件夹内部有一个 index.md 文件，我们需要打开编辑它

设定好 type 以后，我们的主题会自动在这个页面中显示对应的分类，这里主要是说 tags 与 categories 页面，about 页面需要自己在 index.md 中编辑内容。

博客文章编写&检测&发布

说完配置文件以后，接下来就是最关键的一个环节，搭建博客总是需要发布的吧。除了使用 Markdown 语言编写以外，还需要在头部进行对应的设置

在需要发布的 md 文件中增加上述的头部信息后，执行 hexo s 命令，就可以先在本地进行一个预览，没有问题后再进行上传。我们上传的话需要安装自动部署发布工具，同样在终端中执行命令 npm install hexo-deployer-git --save 安装成功后执行 hexo clean && hexo g && hexo d 命令后就表示博客发布成功了，这个命令分别的表示的意思是

  • 清除缓存
  • 生成静态网页
  • 开始部署

接着打开之前创建的 GitHub 源路径就可以查看了。

20180624补充

在发布博客的遇到一个问题，带有图片的博客的上传后图片无法显示出来，后来通过查阅资料得知:

  • 需要将整体 _config.xml 中的 post_asset_folder 设置为 true 默认为 false
  • 执行 npm install hexo-asset-image –save 命令安装一个插件
  • 安装完成后，执行 hexo new 博客名字 后，你会发现你的 _posts 文件夹中多出一个与你博客名字相同的文件夹，将你的图片资源放入后，就可以正常使用原来插入图片的语法了。
  • 执行 hexo g 部署后，前往 public/年/月/日 的文件夹中查看 index.html 文件内的图片资源显示，如果为 <img src="/年/月/日/博客名/图片名称" alt="提示名"> 那么恭喜你，你的博客可以正常显示图片了。
  • 除了这个方法以外，官方文档中推荐使用标签插件 但是我自己尝试了一下好像没有成功。
